# README narrativo di salute: quando è accettabile?

Il confronto cross-repo ha rilevato che `salute/README.md` mescola metodo portabile, filosofia locale e indice KB in forma più narrativa degli altri. È utile come traccia storica ma riduce la leggibilità come bootstrap rapido.

La domanda metodologica non è "correggi salute" — è: quando un README narrativo è una scelta legittima e quando è drift dalla funzione di bootstrap?

## Azione

- leggere `salute/README.md` e classificare ogni sezione: bootstrap operativo / filosofia locale / indice KB / storico-narrativo
- formulare un criterio: la narratività è accettabile se X, è drift se Y
- aggiornare `kb/readme.md` con il criterio, motivato con l'evidenza da salute

## Criterio di chiusura

`kb/readme.md` ha un paragrafo esplicito sulla distinzione tra README narrativo (accettabile in certi domini riflessivi) e README come bootstrap puro. Il criterio è verificabile su casi reali.
